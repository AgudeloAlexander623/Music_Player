## CONTEXTO DEL PROYECTO

**Reproductor** es un clon de Spotify full-stack que agrega resultados de búsqueda desde 6 APIs de música (Spotify, Deezer, YouTube, YouTube Music, MusicBrainz, FMA). Incluye autenticación JWT, favoritos, playlists, reproductor de audio y recomendaciones.

### Stack técnico
| Capa | Tecnología |
|---|---|
| Backend | Node.js + Express 5, ES Modules |
| Frontend | React 19 + Vite 8, react-router-dom v7, CSS puro |
| Base de datos | PostgreSQL 16 con `pg` (pool de conexiones) |
| Microservicio | Python FastAPI (recomendaciones) |
| Infraestructura | Docker + Docker Compose (PostgreSQL, Backend, Frontend con Nginx, Python) |
| Tests | `node:test` en backend, Vitest en frontend |

### APIs externas
| Fuente | Auth | Límites |
|---|---|---|
| Spotify | Client Credentials (Client ID + Secret) | ~1000 req/h |
| YouTube | API Key | 10,000 unidades/día |
| Deezer | Público (sin auth) | 50 req/5s |
| YouTube Music | Usa YouTube API o Invidious (fallback) | Misma cuota que YouTube |
| MusicBrainz | User-Agent header | 1 req/s (por cortesía) |
| FMA (Free Music Archive) | API Key | Sin límite documentado |
| Internet Archive | Público (sin API key) | Sin límite documentado |

## FORMATO DE SALIDA

El documento final debe ser un archivo `.docx` con:

1. **Portada** con nombre del proyecto, materia, semestre, fecha
2. **Índice** de contenidos
3. **Introducción** (2-3 párrafos explicando el sistema)
4. **Cada diagrama en una sección separada** con:
   - Título del diagrama
   - Breve descripción (1-2 párrafos)
   - El diagrama en sí (insertado como tabla, formas, o imagen)
5. **Diagramas profesionales** usando:
   - Formas de Word (rectángulos, rombos, círculos) conectadas con flechas
   - Tablas bien formateadas para casos de uso
   - Colores suaves y consistentes
   - Texto legible (mínimo 10pt)
6. **Pie de página** con nombre del proyecto y número de página
