## CONTEXTO DEL PROYECTO

**Reproductor** es un clon de Spotify full-stack que agrega resultados de búsqueda desde 6 APIs de música (Spotify, MusicBrainz, FMA, YouTube, YouTube Music, Deezer). Incluye autenticación JWT, favoritos, playlists, reproductor de audio y recomendaciones.

### Stack técnico

| Capa | Tecnología |
|---|---|
| Backend | Node.js + Express 5, ES Modules |
| Frontend | React 19 + Vite 8, react-router-dom v7, CSS puro |
| Base de datos | MySQL con mysql2/promise |
| Microservicio | Python FastAPI (recomendaciones) |
| Infraestructura | Docker + Docker Compose (MySQL, Backend, Frontend con Nginx, Python) |
| Tests | `node:test` en backend |

### API externas integradas

| Fuente | Auth | Límites |
|---|---|---|
| Spotify | Client Credentials (Client ID + Secret) | ~1000 req/h |
| YouTube | API Key | 10,000 unidades/día (~50 búsquedas) |
| Deezer | Público (sin auth) | 50 req/5s |
| MusicBrainz | User-Agent header | 1 req/s (por cortesía) |
| FMA (Free Music Archive) | API Key | Sin límite documentado |

### Formato estándar de track (todas las fuentes)

```js
{
  id: string,
  name: string,
  artist: string,
  album: string,
  albumImage: string,    // URL de portada
  previewUrl: string|null, // URL de audio (30s preview)
  duration: number|null,   // en segundos
  source: string           // "spotify" | "musicbrainz" | "fma" | "youtube" | "deezer" | "youtube-music"
}
```

### Endpoints del backend

```
GET  /api/search?q=&page=&limit=     ← Busca en 6 APIs y fusiona
POST /api/auth/register              ← Registro con bcrypt + JWT
POST /api/auth/login                 ← Login con JWT 24h
POST /api/auth/guest                 ← Guest mode (JWT sin BD)
POST /api/auth/verify                ← Verifica validez del token
GET  /api/favorites                  ← Listar favoritos (JWT)
POST /api/favorites                  ← Agregar favorito (JWT)
DELETE /api/favorites/:id            ← Eliminar favorito (JWT)
GET  /api/playlists                  ← Listar playlists (JWT)
POST /api/playlists                  ← Crear playlist (JWT, user real)
PUT  /api/playlists/:id              ← Renombrar (JWT, user real)
DELETE /api/playlists/:id            ← Eliminar (JWT, user real)
GET  /api/playlists/:id/tracks       ← Ver canciones de playlist
POST /api/playlists/:id/tracks       ← Agregar canción (JWT, user real)
DELETE /api/playlists/:id/tracks/:trackId  ← Quitar canción (JWT, user real)
GET  /api/recommendations            ← Proxy a Python
GET  /api/recommendations/status     ← Health check microservicio
```

### Decisiones técnicas importantes

- La BD es **opcional**: si MySQL no está disponible, el backend arranca igual en "modo sin BD". Las funciones de auth, favoritos y playlists fallarán con 500, pero el search funciona.
- El token de Spotify se cachea en memoria y se refresca automáticamente. Hay un mutex para prevenir race conditions.
- MusicBrainz aplica rate limiting de 1.1s entre llamadas con mutex.
- YouTube tiene caché en memoria con TTL de 5 minutos y control de cuota (10k unidades/día).
- La fusión de resultados usa clave `${source}-${id}` para dedup. MusicBrainz se correlaciona con Spotify por nombre/artista normalizado.
- Los tokens JWT de invitados tienen `{ guest: true }` en el payload. Las rutas de escritura (playlists, favoritos) requieren usuario real.
- El frontend usa Axios con interceptor que redirige a `/login` en 401.
