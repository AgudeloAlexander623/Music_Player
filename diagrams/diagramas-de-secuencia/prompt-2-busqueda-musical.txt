Genera el codigo XML de Draw.io para un diagrama de secuencia UML, basado EXCLUSIVAMENTE en el siguiente contenido. No inventes pasos ni interacciones.

IMPORTANTE: XML para importar directo a Draw.io. Estilo "junior": diagrama secuencia clasico pero hecho sin obsesionarse con la perfeccion visual. Colores planos, lineas punteadas sencillas, textos claros y directos. Nada fancy.

Estilo:
- Lifelines: rectangulo fino + linea punteada vertical
- Flechas: lineas con punta de flecha entre lifelines
- Activacion: cajita fina sobre lifeline
- Notas: rectangulo con fondo amarillo clarito si se necesita explicar algo

Colores:
- Lifelines: #ffffff borde #666666
- Flechas: #333333
- Activacion: #dae8fc
- Notas: #fff2cc

Texto: Arial 10pt

--- CONTENIDO ---

LIFELINES:
[Usuario] [Dashboard.jsx] [SearchResults] [api.js] [SearchController] [PluginRegistry] [SpotifyAPI] [MBApi] [FMAAPI] [mergeResults] [MySQL]

FLUJO:
1. Usuario escribe "beatles" → Dashboard.jsx
2. Dashboard.jsx: useEffect detecta >= 2 caracteres, construye URL con AbortController (timeout 10s)
3. Dashboard.jsx → SearchResults: prepara busqueda
4. SearchResults → api.js: GET /api/search?q=beatles&limit=10&page=1
5. api.js → SearchController: request HTTP (con JWT si autenticado)
6. SearchController valida query (>= 2 caracteres)
7. SearchController → PluginRegistry: getAvailable()
8. PluginRegistry → SearchController: return plugins disponibles
9. SearchController → PluginRegistry: searchAll(query, {limit, page})
10. PluginRegistry lanza Promise.all con cada plugin:
    10a. → SpotifyAPI: GET /v1/search?q=beatles
    10b. → MBApi: GET /ws/2/recording/?q=beatles (con rate limit 1s)
    10c. → FMAAPI: GET /api/v1/tracks/search?q=beatles
11. Cada API responde con sus resultados (o error con catch individual)
12. PluginRegistry → SearchController: {results: {spotify: [...], musicbrainz: [...], fma: [...]}, errors: [...]}
13. SearchController → mergeResults: pasa los 3 arrays de tracks
14. mergeResults: deduplica por clave compuesta ${source}-${id}, ordena y combina
15. mergeResults → SearchController: return array unico de tracks
16. SearchController verifica: ¿req.user existe?
    16a. Si → MySQL: INSERT INTO search_history
17. SearchController → api.js: 200 OK {tracks: [...], count: N, sources: {...}, warnings: [...]}
18. api.js → SearchResults: response data
19. SearchResults → Dashboard.jsx: setSearchResults(tracks)
20. Dashboard.jsx renderiza tarjetas de canciones
21. Dashboard.jsx → Usuario: muestra resultados

NOTA: Las 3 APIs se llaman EN PARALELO con Promise.all(). Cada una con su propio try/catch. Si todas fallan → 500. Si al menos una funciona → 200 (o 206 si alguna fallo). MusicBrainz no tiene preview de audio. FMA tiene audio completo (Creative Commons).
