Genera el codigo XML de Draw.io para un diagrama de secuencia UML, basado EXCLUSIVAMENTE en el siguiente contenido.

IMPORTANTE: XML directo para Draw.io. Estilo "junior": diagrama secuencia clasico, simple, colores planos, sin efectos. Hecho por un estudiante que esta aprendiendo UML.

Formato:
- Lifelines: rectangulo + linea punteada vertical
- Flechas con punta entre lifelines
- Activacion: caja fina sobre lifeline
- Separar los 3 flujos

Colores:
- Lifeline: #ffffff borde #666666
- Flechas: #333333 con endArrow=classic
- Activacion: #dae8fc
- Notas: #fff2cc borde #d6b656

Texto Arial 10pt.

--- CONTENIDO DEL FLUJO 1: RECOMENDACIONES PERSONALIZADAS ---

LIFELINES: [Usuario] [Dashboard.jsx] [RecommendationsController] [verifyTokenMW] [python.service.js] [FastAPI] [MySQL]

1. Usuario carga Dashboard → Dashboard.jsx
2. Dashboard.jsx: useEffect al montar componente
3. Dashboard.jsx → RecommendationsController: GET /api/recommendations (con JWT)
4. RecommendationsController → verifyTokenMW: verifica token
5. verifyTokenMW → RecommendationsController: next() (valido)
6. RecommendationsController → python.service.js: getRecommendations(userId, limit)
7. python.service.js → MySQL: fetchUserTracks(userId) — busca favoritos del usuario (findMany favorite_tracks, si no hay, busca search_history)
8. MySQL → python.service.js: return tracks
9. python.service.js → FastAPI: POST http://localhost:8000/recommend {tracks: [...], count: 10}
10. FastAPI: ejecuta TF-IDF / content-based scoring contra catalogo interno (60 tracks)
11. FastAPI → python.service.js: return {recommendations: [{id, name, artist, score, reason}], metadata}
   (Si falla conexion a Python: usa FALLBACK_RECOMMENDATIONS hardcodeadas)
12. python.service.js → RecommendationsController: return {success, recommendations}
13. RecommendationsController → Dashboard.jsx: 200 {recommendations: [...]}
14. Dashboard.jsx renderiza <ContentSection title="Recomendado para ti" />
15. Dashboard.jsx → Usuario: ve las recomendaciones

--- CONTENIDO DEL FLUJO 2: HEALTH CHECK ---

LIFELINES: [Usuario] [RecommendationsController] [python.service.js] [FastAPI]

1. Usuario o sistema: GET /api/recommendations/status → RecommendationsController
2. RecommendationsController → python.service.js: checkPythonServiceHealth()
3. python.service.js → FastAPI: GET /health (timeout 3s)
4. FastAPI → python.service.js: {status: "ok", catalog_size: 60, version: "0.1.0-beta"}
5. python.service.js → RecommendationsController: return {success, data}
6. RecommendationsController → Usuario: 200 {success, status, ...}

--- CONTENIDO DEL FLUJO 3: RECOMENDACIONES POPULARES ---

LIFELINES: [Usuario] [RecommendationsController] [python.service.js] [FastAPI]

1. Usuario: GET /api/recommendations/popular → RecommendationsController (sin auth)
2. RecommendationsController → python.service.js: getPopularRecommendations(limit)
3. python.service.js → FastAPI: GET /recommend/popular?count=10
4. FastAPI: random subset del catalogo (catalog.py)
5. FastAPI → python.service.js: return tracks
6. python.service.js → RecommendationsController: return {success, recommendations}
7. RecommendationsController → Usuario: 200 OK

NOTA: Si el usuario tiene favoritos, las recomendaciones se basan en similitud (artista, genero, palabras clave). Si no tiene favoritos, devuelve canciones populares aleatorias. Si Python no responde, usa fallback hardcodeado en Node.
