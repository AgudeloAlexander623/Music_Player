--- CONTENIDO DEL FLUJO 1: FLUJO GENERAL COMPLETO ---

LIFELINES (en este orden):
[Usuario] [React App] [Router] [Pagina] [api.js] [Express] [Middleware] [Controller] [Service] [DB/API]

PASOS:
1. Usuario abre la app → React App
2. React App: main.jsx carga App.jsx (BrowserRouter + AuthProvider)
3. React App: AuthContext.verifySession() revisa si hay token en localStorage
   3a. ¿Token? → POST /auth/verify → Express → Middleware → Controller → ok
   3b. ¿No token? → setUser(null)
   3c. ¿Guest? → setUser({guest:true})
4. React App renderiza: <Topbar /> + <Sidebar /> + <Outlet /> (pagina activa)
5. React App → Usuario: ve la interfaz
6. Usuario escribe "queen" en buscador → Pagina (Dashboard.jsx)
7. Pagina: handleSearch() construye URL con AbortController (timeout 10s)
8. Pagina → api.js: GET /api/search?q=queen&page=1&limit=10
9. api.js: interceptor request adjunta Bearer token de localStorage si existe
10. api.js → Express: HTTP request
11. Express: app.js recibe request
12. Express: CORS middleware permite origen del frontend
13. Express: JSON body parser
14. Express: Rate limiter (30 req/min)
15. Express → Router: /api/search
16. Router → Middleware: verifyToken (si hay Bearer, decodifica JWT, setea req.user)
17. Middleware → Controller: searchController(req, res)
18. Controller → Service: llama servicios (pluginRegistry.searchAll)
19. Service → DB/API: consulta Deezer, YouTube, FMA, MusicBrainz, IA (en paralelo)
20. DB/API → Service: return resultados (o errores con catch)
21. Service: mergeResults() deduplica y ordena
22. Service → Controller: return tracks combinados
23. Controller: si req.user existe → INSERT search_history en DB
24. Controller → Middleware: JSON response
25. Middleware → Router → Express: 200 OK
26. Express → api.js: response {tracks: [...], count: N}
27. api.js → Pagina: data
28. Pagina: setSearchResults(tracks), renderiza <SearchResults />
29. Pagina → Usuario: ve las tarjetas de canciones

--- CONTENIDO DEL FLUJO 2: FLUJO DE AUTENTICACION Y AUTORIZACION ---

LIFELINES:
[Frontend Component] [api.js Interceptor] [Express] [verifyTokenMW] [requireRealUser] [Controller]

PASOS:
1. Componente hace request que requiere auth → api.js Interceptor
2. api.js Interceptor: si hay token en localStorage → adjunta header Authorization: Bearer <token>
3. api.js Interceptor → Express: request con header Auth
4. Express → verifyTokenMW: ejecuta middleware
5. verifyTokenMW: jwt.verify() — valido: req.user = decoded; expirado/invalido: 401
6. verifyTokenMW → Express: next()
7. Express → requireRealUser: si aplica (endpoints de escritura)
8. requireRealUser: ¿req.user.guest? Si → 403; No → next()
9. requireRealUser → Express: next()
10. Express → Controller: ejecuta logica de negocio
11. Controller procesa y responde
12. Controller → Express → api.js Interceptor: response
13. api.js Interceptor: ¿response 401? → borra localStorage, redirige a /login
14. api.js Interceptor → Componente: data

NOTA: Las capas del sistema son:
  CAPA 1 - Presentacion: React (Pages, Components, Context)
  CAPA 2 - API Gateway: Express (CORS, JSON, Rate Limit, Routers)
  CAPA 3 - Logica: Middleware, Controllers, Services, Utils
  CAPA 4A - Datos: PostgreSQL (users, favorite_tracks, playlists, playlist_tracks, search_history)
  CAPA 4B - Externas: Deezer, YouTube, MusicBrainz, FMA, Internet Archive, Python FastAPI
