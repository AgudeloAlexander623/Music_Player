FLUJO DE AUTENTICACION Y AUTORIZACION
================================================================================

[Frontend Component]   [api.js Interceptor]   [Express]   [verifyTokenMW]   [requireRealUser]   [Controller]
    |                        |                    |             |                 |                 |
    | 1. Cualquier request   |                    |             |                 |                 |
    | que requiera auth      |                    |             |                 |                 |
    |----------------------->|                    |             |                 |                 |
    |                        |                    |             |                 |                 |
    |                        | 2. Interceptor     |             |                 |                 |
    |                        | request:           |             |                 |                 |
    |                        | Si hay token en    |             |                 |                 |
    |                        | localStorage       |             |                 |                 |
    |                        | -> header:         |             |                 |                 |
    |                        | Authorization:     |             |                 |                 |
    |                        |   Bearer <token>   |             |                 |                 |
    |                        |                    |             |                 |                 |
    |                        | 3. Request con     |             |                 |                 |
    |                        | header Auth        |             |                 |                 |
    |                        |------------------->|             |                 |                 |
    |                        |                    |             |                 |                 |
    |                        |                    | 4. verifyToken            |                 |
    |                        |                    |------------>|                 |                 |
    |                        |                    |             |                 |                 |
    |                        |                    |             | 5. jwt.verify() |                 |
    |                        |                    |             | - Valido:       |                 |
    |                        |                    |             |   req.user =    |                 |
    |                        |                    |             |   decoded       |                 |
    |                        |                    |             | - Expirado: 401 |                 |
    |                        |                    |             | - Invalido: 401 |                 |
    |                        |                    |             |                 |                 |
    |                        |                    |             | 6. next()       |                 |
    |                        |                    |<------------|                 |                 |
    |                        |                    |             |                 |                 |
    |                        |                    | 7. requireRealUser (si aplica)              |
    |                        |                    |------------|--------------->|                 |
    |                        |                    |             |                 |                 |
    |                        |                    |             | 8. req.user.guest?              |
    |                        |                    |             | - Si -> 403     |                 |
    |                        |                    |             | - No -> next()  |                 |
    |                        |                    |<------------|---------------|                 |
    |                        |                    |             |                 |                 |
    |                        |                    | 9. Controller ejecuta logica              |
    |                        |                    |------------|---------------|--------------->|
    |                        |                    |             |                 |                 |
    |                        | 10. Respuesta     |             |                 |                 |
    |                        |<-------------------|             |                 |                 |
    |                        |                    |             |                 |                 |
    |                        | 11. Interceptor    |             |                 |                 |

    |                        |                    |             |                 |                 |

DIAGRAMA DE CAPAS (LAYERS)
================================================================================

   CAPA 1: PRESENTACION (Frontend)
   +-----------------------------------------------------------------+
   | React 19 + Vite                                                 |
   | - Pages: Login, Dashboard, Favorites, Playlists, Profile        |
   | - Components: Topbar, Sidebar, Player, SearchResults            |
   | - Context: AuthContext (estado global de usuario)               |
   | - Services: api.js (Axios con interceptors)                     |
   +-----------------------------------------------------------------+
            |  HTTP/JSON (localhost:4000/api)
            ▼
   CAPA 2: API GATEWAY (Backend)
   +-----------------------------------------------------------------+
   | Express 5                                                       |
   | - CORS middleware                                               |
   | - JSON body parser                                              |
   | - Rate limiters (por ruta)                                      |
   | - Routers (auth, search, favorites, playlists, recommendations) |
   +-----------------------------------------------------------------+
            |
            ▼
   CAPA 3: LOGICA DE NEGOCIO
   +-----------------------------------------------------------------+
   | Middleware: verifyToken, requireRealUser                        |
   | Controllers: Auth, Search, Favorites, Playlists, Recommendations|
   | Services: auth, python, mergeResults                            |
   | Utils: logger, validateEnv                                      |
   +-----------------------------------------------------------------+
            |
            ├--------------------------┐
            ▼                          ▼
   CAPA 4A: DATOS             CAPA 4B: EXTERNAL
   +-------------------+     +-------------------------+
   | PostgreSQL        |     | Deezer API              |
   | - users           |     | YouTube API             |
   | - favorite_tracks |     | MusicBrainz API         |
   | - playlists       |     | FMA API                 |
   | - playlist_tracks |     | Internet Archive API    |
   | - search_history  |     | Python FastAPI (recom)  |
   +-------------------+     +-------------------------+


================================================================================
RESUMEN DE FLUJO DE DATOS
================================================================================

   Usuario
      ↓ (click, input)
   Componente React (Pagina)
      ↓ (llamada a api.js)
   Interceptor api.js
      ↓ (adjunta JWT si existe)
   HTTP Request (JSON)
      ↓
   Express app.js
      ↓ (CORS → JSON parse → Rate Limit → Router)
   Ruta específica
      ↓ (middleware si aplica: verifyToken → requireRealUser)
   Controller
      ↓ (orquesta: valida → llama servicios → responde)
   Service / External API
      ↓
   Database / API Response
      ↑ (camino inverso con JSON)
   Controller responde
      ↑
   api.js recibe respuesta
      ↑ (interceptor: si 401 → logout)
   Componente actualiza estado
      ↑ (setState / context)
   React re-renderiza UI
      ↑
   Usuario ve el resultado
