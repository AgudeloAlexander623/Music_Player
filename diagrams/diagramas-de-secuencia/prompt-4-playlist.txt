Genera el codigo XML de Draw.io para un diagrama de secuencia UML, basado EXCLUSIVAMENTE en el siguiente contenido.

IMPORTANTE: XML directo para Draw.io. Estilo "junior": diagrama simple, colores planos, sin rebuscar. Que se vea hecho por un estudiante de software.

Formato clasico de secuencia:
- Lifelines: rectangulo + linea punteada vertical
- Flechas con punta entre lifelines
- Activacion: caja fina sobre lifeline
- Separar los 4 flujos con espacio

Colores planos:
- Lifeline: #ffffff borde #666666
- Flechas: #333333
- Activacion: #dae8fc
- Notas: #fff2cc borde #d6b656

Texto Arial 10pt.

--- CONTENIDO DEL FLUJO 1: CREAR PLAYLIST ---

LIFELINES: [Usuario] [Playlists.jsx] [PlaylistsController] [verifyTokenMW] [requireRealUser] [MySQL]

1. Usuario llena nombre y descripcion → Playlists.jsx
2. Playlists.jsx: click "Crear Playlist"
3. Playlists.jsx → PlaylistsController: POST /api/playlists {name, description}
4. PlaylistsController → verifyTokenMW: verifica JWT
5. verifyTokenMW → PlaylistsController: next()
6. PlaylistsController → requireRealUser: bloquea guest
7. requireRealUser → PlaylistsController: next()
8. PlaylistsController valida name no vacio
9. PlaylistsController → MySQL: INSERT INTO playlists (user_id, name, description) VALUES (...)
10. MySQL → PlaylistsController: RETURNING id
11. PlaylistsController → Playlists.jsx: 201 {success, playlist}
12. Playlists.jsx agrega a la lista
13. Playlists.jsx → Usuario: ve la nueva playlist

--- CONTENIDO DEL FLUJO 2: AGREGAR TRACK A PLAYLIST ---

LIFELINES: [Usuario] [SearchResults] [PlaylistsController] [verifyTokenMW] [requireRealUser] [MySQL]

1. Usuario click + en cancion, selecciona playlist del dropdown → SearchResults
2. SearchResults → PlaylistsController: POST /api/playlists/:id/tracks {external_track_id, source, ...}
3. PlaylistsController → verifyTokenMW + requireRealUser: verifican
4. PlaylistsController → MySQL: findOne('playlists', {id: playlistId})
5. MySQL → PlaylistsController: return playlist
6. PlaylistsController verifica ownership: playlist.user_id == req.user.userId? No → 403
7. PlaylistsController valida track fields
8. PlaylistsController → MySQL: INSERT INTO playlist_tracks (...)
9. MySQL → PlaylistsController: RETURNING id
10. PlaylistsController → SearchResults: 201 {success, track}
11. SearchResults → Usuario: toast "Agregada a playlist"

--- CONTENIDO DEL FLUJO 3: LISTAR PLAYLISTS ---

LIFELINES: [Usuario] [Playlists.jsx] [PlaylistsController] [verifyTokenMW] [MySQL]

1. Usuario navega a /playlists → Playlists.jsx
2. Playlists.jsx: loadPlaylists() → GET /api/playlists
3. Playlists.jsx → PlaylistsController: request
4. PlaylistsController → verifyTokenMW: verifica (guest puede ver, pero vacio)
5. verifyTokenMW → PlaylistsController: next()
6. PlaylistsController → MySQL: findMany('playlists', {user_id: userId})
7. MySQL → PlaylistsController: SELECT * FROM playlists WHERE user_id = ?
8. PlaylistsController → Playlists.jsx: 200 {playlists: [...]}
9. Playlists.jsx renderiza lista
10. Playlists.jsx → Usuario: ve sus playlists

--- CONTENIDO DEL FLUJO 4: ELIMINAR PLAYLIST ---

LIFELINES: [Usuario] [Playlists.jsx] [PlaylistsController] [verifyTokenMW] [requireRealUser] [MySQL]

1. Usuario click eliminar → Playlists.jsx
2. Playlists.jsx → PlaylistsController: DELETE /api/playlists/:id
3. PlaylistsController → verifyTokenMW + requireRealUser: verifican
4. PlaylistsController: busca playlist y verifica ownership
5. PlaylistsController → MySQL: DELETE FROM playlists WHERE id = ? (cascada: tracks se borran solos)
6. MySQL → PlaylistsController: ok
7. PlaylistsController → Playlists.jsx: 200 {success}
8. Playlists.jsx elimina de la lista
9. Playlists.jsx → Usuario: ve lista actualizada

NOTA: Solo usuarios registrados pueden crear/editar/eliminar playlists (requireRealUser → 403 si guest). Los GET permiten guest pero devuelven lista vacia.
